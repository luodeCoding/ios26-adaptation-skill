## 图 1：封面图

**视觉目标**：第一眼看到"iOS 26"四个大字，第二眼看到 Xcode 代码，其他都是氛围。

```
生成一张科技博客封面图，横版 16:9。

【画面最核心 / 占 C 位】
画面正中央偏上，有一个巨大的发光数字"26"，字体粗大、带有金属玻璃质感，发光颜色是电光蓝到青色的渐变，占据画面约 25% 的高度。这是全图最亮、最大的元素，必须一眼看到。

【第二焦点 / 画面下半部分】
数字"26"的正下方，是一台打开的 MacBook Pro，屏幕显示 Xcode 代码编辑器界面，代码占满整个屏幕，清晰可见。电脑屏幕是画面中第二亮的区域。电脑以 15 度角微微朝向观众，有真实的金属质感和柔和阴影。

【背景氛围 / 不抢戏】
背景是极深的蓝黑色（接近纯黑），有非常淡的粒子光点和细线条组成的网络网格，像数字空间的底纹。背景亮度必须远低于前景，不能分散注意力。

【点缀元素 / 很小、很淡】
- 画面左上角：一个极小的 Swift 语言图标（鸟类形状），半透明
- 画面右下角：几个极小的代码符号 `{ }` `[ ]` 漂浮，半透明
- 画面边缘：非常淡的蓝色光晕，像镜头光晕

【绝对不要】
不要标题文字、不要中文、不要 paragraph 文字、不要多余装饰。画面要干净，只有"26"+ Xcode + 深色底。

风格：苹果发布会级简洁，深色模式，8K 渲染，电影级光影，高对比度。
```

## 图 2：项目架构图

**视觉目标**：一眼看出 4 个方块，每个方块里的图标清晰可辨。

```
生成一张信息图，横版 16:9，深色背景接近纯黑。

【主体 / 4 个等大方块】
画面正中央，均匀排列 4 个同样大小的圆角矩形方块，每个方块约占画面宽度的 18%、高度的 40%。4 个方块横向等距排成一排，处于画面黄金分割位置（偏上）。

方块 1（最左）：深橙色背景，中央有一个白色的放大镜图标，图标很大，占方块高度的 50%。
方块 2（左中）：深蓝色背景，中央有一个白色的文档/页面图标，图标很大，占方块高度的 50%。
方块 3（右中）：深绿色背景，中央有一个白色的书本/指南图标，图标很大，占方块高度的 50%。
方块 4（最右）：深紫色背景，中央有一个白色的循环箭头/同步图标，图标很大，占方块高度的 50%。

【连接关系】
4 个方块之间有极细的白色半透明线条连接，线条带微弱发光。线条很细，不能粗过 2 像素，不能抢戏。

【背景】
纯黑背景，方块下方有非常淡的倒影，像放在黑色玻璃桌面上。背景中没有任何其他元素。

【绝对不要】
不要文字、不要标题、不要说明标签、不要数字编号。只留 4 个彩色方块 + 白图标 + 连接线 + 倒影。

风格：极简、苹果设计奖级质感、高对比度、每一个像素都有目的。
```

## 图 3：Scanner 扫描演示图

**视觉目标**：左边乱 → 右边治，对比强烈，一眼看懂"扫描"在做什么。

```
生成一张分屏对比图，横版 16:9，深色背景。

【左半屏 / 约 45% 宽度】
显示一段杂乱的 iOS 代码，像 Xcode 深色主题。
- 代码行数约 15-20 行
- 其中 5 处有明显标记：红色下划线 + 黄色高亮背景，表示"错误"
- 2 处有黄色高亮背景，表示"警告"
- 代码整体看起来"脏"、拥挤、有问题
- 屏幕边缘有微微的暗角，暗示"老旧/待修复"

【右半屏 / 约 45% 宽度】
显示清理后的扫描结果面板，整洁有序。
- 顶部有一条绿色的进度条，显示"100%"
- 下方是 3 个清晰的区块，用颜色区分：
  - 红色区块：错误列表，带 X 图标
  - 橙色区块：警告列表，带 ! 图标
  - 蓝色区块：信息列表，带 i 图标
- 每个列表约 3-4 行，简洁干净
- 整体看起来"新"、"清晰"、"已解决"

【中间分隔 / 约 10% 宽度】
中间有一条垂直的发光分界线，颜色从左侧的暗红渐变到右侧的亮绿。分界线中央有一个发光的扫描光标/光束图标，暗示"扫描"动作从左到右发生。

【绝对不要】
不要真实可读的文字（用模拟代码），不要中文，不要多余的 UI 元素。左右对比要强烈、直观。

风格：开发者工具截图风格，高对比度，信息设计清晰。
```

## 图 4：适配路线图

**视觉目标**：从左到右，一眼看出"两步走"，橙色紧急 → 蓝色规划。

```
生成一张两阶段流程图，横版 16:9，深色背景。

【核心 / 一条发光时间线】
画面中央偏下，有一条从左到右的水平粗线条，约画面高度的 8%。线条是渐变色：最左端是亮橙色，向右逐渐过渡到亮蓝色，最右端是深蓝色。线条有发光效果，像霓虹灯管。这是全图的视觉引导线。

【第一阶段卡片 / 左侧】
时间线左端上方，有一个较大的圆角矩形卡片（约占画面宽度 25%、高度 35%）。
- 卡片边框和内部强调色是橙色
- 卡片内最显眼的位置：一个白色的扳手/工具图标，很大（占卡片高度 40%）
- 卡片下方有一个小圆点连接在时间线上，圆点是橙色发光
- 卡片整体有微微向上浮起的感觉（阴影暗示）

【第二阶段卡片 / 右侧】
时间线右端上方，有一个同样大小的圆角矩形卡片。
- 卡片边框和内部强调色是蓝色
- 卡片内最显眼的位置：一个白色的画笔/设计图标，很大（占卡片高度 40%）
- 卡片下方有一个小圆点连接在时间线上，圆点是蓝色发光
- 同样有浮起感

【背景】
极深色背景（接近纯黑），时间线下方有非常淡的网格线。画面左侧边缘有极淡的橙色光晕，右侧边缘有极淡的蓝色光晕，呼应两个阶段的色调。

【绝对不要】
不要文字、不要日期标注、不要箭头符号。用颜色（橙→蓝）和位置（左→右）表达时间先后。

风格：极简信息图，苹果 Keynote 发布会风格，高级感，每一个元素都在讲故事。
```

## 使用方式

1. **整段复制**，不要只复制一部分
2. **一张一张发**，等出图了再发下一段
3. **如果 AI 没听清楚主次**：直接告诉它"把 26 放大到画面 1/3""代码再明显一点"
4. **出图后自己加字**：用 Figma / Keynote / Canva 加标题，比让 AI 生成文字靠谱 100 倍
